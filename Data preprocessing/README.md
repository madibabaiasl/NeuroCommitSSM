# NeuroCommitSSM Data Preprocessing

This folder contains the preprocessing notebooks used in the **NeuroCommitSSM** pipeline. These notebooks prepare the raw multimodal recordings for downstream modeling by cleaning, synchronizing, labeling, validating, and exporting the data into training-ready formats.

## Overview

The preprocessing pipeline is organized into sequential phases. Each phase handles a specific part of the workflow, starting from early-stage signal preparation and continuing through labeling, quality checking, and export generation.

These notebooks support the preparation of multimodal data collected from:

- EEG
- EMG
- Eye-tracking

The outputs of these phases are used for model training, validation, and inference in the NeuroCommitSSM framework.

## Purpose

These notebooks are provided to:

- document the full preprocessing workflow of the project
- prepare raw multimodal recordings for analysis
- create synchronized and labeled data
- support reproducibility of the NeuroCommitSSM pipeline
- generate structured outputs for model development and evaluation

## Recommended Workflow

The notebooks are generally intended to be run in phase order:

1. `Phase-1B.ipynb`
2. `Phase-1C.ipynb`
3. `Phase-2A.ipynb`
4. `Phase-2B.ipynb`
5. `Phase-3.ipynb`
6. `Phase-4A.ipynb`
7. `Phase-4B.ipynb`
8. `Phase-5A.ipynb`
9. `Phase-5B.ipynb`

Running them in sequence helps ensure that each stage receives the outputs generated by the previous stage.

## Usage

Open the notebooks in Jupyter Notebook or JupyterLab and run them according to the required pipeline stage.

These notebooks may be useful for:

- reproducing the preprocessing workflow
- understanding how raw recordings are converted into usable datasets
- validating intermediate processing outputs
- preparing project data for training, evaluation, or inference

## Notes

- The notebooks are part of the NeuroCommitSSM preprocessing pipeline.
- Some notebooks may depend on outputs generated by earlier phases.
- Users should follow the intended phase order for correct execution.
- Specific file paths, dataset locations, and dependencies may need to be updated based on the local environment.

## Summary

This folder contains the notebook-based preprocessing workflow for NeuroCommitSSM. Together, these files define the major stages used to convert raw multimodal recordings into synchronized, labeled, validated, and model-ready datasets for downstream learning and inference.
